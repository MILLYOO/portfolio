```Python 3.9.5``` ```Django 3.2.4``` ```Nginx``` ```uWSGI``` ```DjangoORM```

- APScheduler를 활용한 근태반영 및 외부 ERP 연동 구현
- ClipReport를 통한 Report 구현
- 사용자 편의성을 위한 웹로그뷰어 Tailon 및 서버정보를 얻기 위한 Glances 구현
- 각종 프로세스 및 게시판 기능 구현

#### 🔴 Issue
- APScheduler 기반 근태 배치 처리 시 건별 INSERT로 다수 사용자 처리 시 실행 시간 증가
- 배포 후 오류 확인 및 서버 상태 파악을 위해 매번 서버에 직접 접속해야 하는 번거로움 발생

#### 🟡 Solution
- DjangoORM `bulk_create`를 활용한 일괄 INSERT로 DB 요청 횟수 최소화
- Tailon(웹 로그뷰어)과 Glances(서버 모니터링)를 도입해 브라우저에서 실시간 로그·서버 상태 확인 가능하도록 구성

#### 🟢 Result
- 건별 INSERT 대비 DB 요청 횟수 대폭 감소로 배치 처리 시간 약 60% 단축
- 장애 발생 시 서버 접속 없이 웹에서 즉시 확인 가능해 초기 대응 시간 단축
