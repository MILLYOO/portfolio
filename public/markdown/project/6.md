```Java``` ```Spring Boot``` ```Spring Batch``` ```Spring Security``` ```AOP``` ```Kubernetes``` ```MySQL``` ```MyBatis```

#### 🔴 Issue
- 상시모니터링 데이터(IT리스크·KRI)를 수기로 취합·가공해 내부 DB에 적재하는 방식으로 인해 적재 지연 및 데이터 정합성 오류 발생
- Quartz 스케줄러 기반 상시 서버 운영으로 배치 미실행 구간에도 서버 자원 100% 점유
- 리스크 시나리오별 위험도 책정 기준 부재로 담당자마다 일관성 없는 관리
- 데이터 접근 권한 제어 미흡으로 타 사용자 데이터 접근 가능한 IDOR 취약점 존재

#### 🟡 Solution
- 상시모니터링 데이터를 내부 DB 구조에 맞게 가공·정제하는 **ETL 배치 설계 및 개발 전담** (매일 자동 적재)
- Quartz 상시 서버 구조를 **Kubernetes CronJob + CommandLineRunner** 방식으로 전환, 배치 실행 시에만 Pod 기동되도록 아키텍처 개선
- 상시모니터링·위험도 책정 데이터가 감사인 **승인·반려 워크플로우**를 통해서만 반영되도록 결재 프로세스 설계 및 구현
- **커스텀 어노테이션 + AOP** 기반 IDOR 방어 로직 구현, 데이터 소유권 검증을 공통 처리하여 API 레벨에서 타 사용자 데이터 접근 차단

#### 🟢 Impact
- 상시모니터링 데이터 적재 소요시간 **수 시간 → 익일 자동 적재**로 단축
- 배치 미실행 구간 서버 자원 점유 **약 70% 절감**
- 수기 데이터 취합 업무 **주 8시간 → 완전 자동화**
- 감사인 승인 워크플로우 도입으로 리스크 데이터 변경 이력 **100% 추적** 가능
- 커스텀 어노테이션 기반 소유권 검증으로 **IDOR 취약점 완전 제거**, 보안성 검토 통과
